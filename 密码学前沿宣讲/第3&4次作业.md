# 1.简述你对可信执行环境TEE的理解，包括优点、可能存在的问题等等。
1.TEE：早在2009年，OMTP就在《OMTP ADVANCED TRUSTED ENVIRONMENT: OMTP TR1 v1.1》中定义了执行环境EE与可信执行环境TEE。执行环境EE是一组硬件和软件组件，提供计算、内存管理、输入/输出等支持应用程序所需的设施，通常由以下核心要素组成：处理单元，处理单元与代码、指令、数据、资源之间的一组连接，存储数据和代码的物理内存，启动过程初始化机制，代码、EE本身的数据集和密钥等。在此基础上，文档中设定了Profiles1和Profiles2这两套安全标准，只要一个EE满足其中一套标准，就可以被称为可信执行环境TEE。这两个Profiles主要规定了TEE应该具备的功能性和抗攻击范围，其中Profiles1提供了基础级别的安全保护，而Profiles2提供了更高级别、更强的安全保护。TEE是计算设备中一个受硬件隔离和加密保护的区域，其内部的代码与外部的普通执行环境完全隔离，能够保护可信应用程序的安全执行环境，从而确保其机密性、完整性和认证性。尽管TEE提供了强大的安全保障，但是对于极度敏感的代码或服务，某些应用程序可能会选择在通用集成电路卡UICC中运行，因为UICC比TEE的物理安全性更高。虽然TEE的概念由OMTP提出，但GlobalPlatform在TEE规范的全球推广中发挥了较为关键的作用。从2010年起，GP就正式提出了TEE概念，并在转年开始制定一系列涵盖应用接口、安全存储、身份认证的国际标准。目前国际上大多数基于TEE技术的Trust OS都遵循GP规范，GP建立的检测认证体系也成为了行业通用的准绳。
2.TEE的优点：与安全多方计算MPC和同态加密等计数相比，TEE结合了底层密码学基础与硬件及系统安全的上层实现，具有较高的运算效率和较低的损耗，因为TEE是在CPU内部进行铭文计算，即数据只在内存中加密，进入CPU核心后再解密，因此其计算速度较高，且整体性能损耗低，可以支持大规模数据量和高频实时计算；而传统的公钥密码学由于涉及大量的加密运算和通信开销，通常会带来极大的性能损耗。而且被加密的数据只有在TEE内部才会被解密计算，计算完成后只输出结果，不会暴露原始明文数据，安全性较高。此外，TEE环境通常支持标准的编程语言，对开发者较为友好，可以支持多层次、高复杂度的算法逻辑，省去了将业务逻辑重写为特定形式的成本。而且用户可以通过TEE的远程认证机制来检测代码是否被篡改以及底层硬件环境是否真是可信，防止攻击者通过修改代码来窃取数据，从而有效地抵御了恶意敌手的问题。
3.TEE可能存在的问题：由于TEE并不像MPC那样基于数学原理来保证安全，而是依赖Intel、ARM、AMD等芯片厂商，其软件方案的实现与底层硬件架构高度绑定，难以在不同的硬件平台间迁移，且难以保证硬件制造商的硬件设计没有逻辑上的缺陷，而一旦底层硬件漏洞，需要通过升级固件或更换CPU等操作来解决，在大规模数据场景下造成了高昂的维护成本。此外，TEE虽然隔离了软件层的攻击，但仍面临测信道攻击的威胁。攻击者虽然无法直接读取内存，但是可以通过观察CPU的缓存、电压、执行时间等物理特征来推断出安全区的私钥和数据，对安全性造成了极大的威胁。
# 2.简述人工智能技术在公钥密码学分析中的应用情况。
总体来看，人工智能技术在公钥密码学中的辅助功能较为强大。以相对更成熟的人工智能在测信道分析中的应用为例。传统的公钥算法主要基于循环结构的运算，比如RSA基于大整数的模幂，ECC基于椭圆曲线标量乘法。这些传统方法对波形对齐和噪声的要求较高，而人工智能使得公钥密码的测信道分析从依赖于攻击者精通信号处理数学转变为依赖于数据采集质量和模型训练等能力，从而在降低攻击门槛的同时提高了攻击的成功率。首先，攻击者采集大量带有已知密钥的原始波形，将其作为数据集用于训练CNN等深度学习网络的提取特征和降噪的能力中，这样当攻击者在不知道密钥的情况下获取到目标设备时，可以通过采集其运行加密时的波形并输入给之前训练好的深度学习网络，从而获取由模型推测出的密钥概率。通过借助深度学习网络的识别能力，可以克服波形抖动带来的对齐特征点的困难，而无需花费人工重对齐波形的精力。而且深度学习网络的函数逼近能力较强，能够在没有明确指示的情况下在训练中自动进行高阶分析，从而抵消掩码的作用而直接恢复原始密钥。除此之外，传统的公钥算法的循环结构运输的序列操作时间较长，使得当前的密钥比特处理可能会受到上一个比特的状态的影响；而LSTM等深度学习网络对这方面的处理能力较强，而且可以通过识别出操作序列的语法，从而推断出私钥的比特流。

# 4.我们在对比同类型的密码协议优劣时，可以从哪几个角度去进行对比？为什么需要考虑这些角度？
首先，在对比同类型的密码协议优劣时，需要考虑协议的安全性。一般而言，会更加倾向基于DDH假设等标准假设构建的协议，而不是其它假设，因为基于标准假设构建的协议已经经过了数十年的攻击验证，安全性更高。除了假设的安全性，算法依赖的数学计算的复杂性对于安全性也非常重要，越复杂难解的计算安全性越高。此外，不可取分性也是衡量安全性的一大因素。比如选择明文攻击下的不可区分性要求即使攻击者能加密任何消息也无法区分密文对应哪一条明文，选择密文攻击下的不可区分性要求即使攻击者能请求解密也能够保持安全。
除此之外，密码协议的效率和性能也是一个很重要的衡量因素。比如，基于大整数分解的RSA算法的密钥生成速度和解密速度较慢，而基于椭圆曲线标量乘法的ECC算法的计算量就更小一些，因此其计算开销也就更小一些。而对于通信的开销，一般往返次数越多。比如，对于TLS 1.2而言，需要2次RTT才能完成握手，而TLS 1.3只需1次或不需RTT就可以，相比1.2而言实现了显著的性能提升，有利于用户体验。而且协议在运行时所需的动态内存也值得考量，比如，相比需要服务器为每个连接维护大量状态信息的协议而言，Cookie机制等轻量级状态的协议需要更少的内存，在大规模应用的场景下的性能也就更优。
而密码协议的实用性也非常重要，如果一个密码协议难以在实际应用中进行部署，那么即使其结构再精妙也不实用。比如，如果一个协议没有经过审计的现成的开源库支持，而是需要开发者自己来编写加密代码，可能会引入安全漏洞，而且会耗费大量成本。此外，对PKI依赖度较高的协议的实用性相对而言不是很高，比如TLS依赖于CA体系，维护成本较高，而SSH通常使用TOFU或者手动分发公钥，相比之下不需要庞大的PKI的支持，对小型系统而言实用性更高。
